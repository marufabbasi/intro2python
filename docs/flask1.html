Install Ubuntu in VirtualBox, Set Up Nginx, and Host a Flask App

A practical end-to-end guide for creating an Ubuntu virtual machine, installing Nginx, deploying a simple Flask application, and serving it through Nginx using Gunicorn.

✅ 1. Requirements

VirtualBox
Install Oracle VirtualBox on your host machine.
Ubuntu ISO
Download an Ubuntu Desktop or Server ISO.
Resources
Recommended: 2 CPUs, 4 GB RAM, 25 GB disk.
Internet
Needed inside the VM for package installation.

This guide uses Ubuntu commands. It works well with Ubuntu Server because it is lightweight, but Ubuntu Desktop is also fine.

🖥️ 2. Create the Ubuntu VM in VirtualBox

  1. Open VirtualBox.
  2. Click New.
  3. Name the VM: Ubuntu-Flask-Nginx.
  4. Select the downloaded Ubuntu ISO.
  5. Allocate at least 4096 MB RAM.
  6. Allocate at least 2 CPUs.
  7. Create a virtual disk of at least 25 GB.
  8. Finish and start the VM.

Tip: If your machine has limited memory, 2 GB RAM can work for Ubuntu Server.

🐧 3. Install Ubuntu

  1. Choose Try or Install Ubuntu.
  2. Select your language and keyboard layout.
  3. Use default storage options unless you need custom partitions.
  4. Create a username and password.
  5. Complete installation and reboot.

After rebooting, remove the ISO from the virtual optical drive if VirtualBox asks.

🌐 4. Configure VM Networking

For easy access from your host machine, use Bridged Adapter or NAT with port forwarding.

Option A: Bridged Adapter

  1. Shut down the VM.
  2. Go to Settings → Network.
  3. Set Attached to as Bridged Adapter.
  4. Start the VM.
Find VM IP address 
ip addr show

Look for an IP like 192.168.x.x. You will use it in your browser.

Option B: NAT Port Forwarding

Forward host port 8080 to guest port 80.

  1. VM Settings → Network → Advanced → Port Forwarding.
  2. Add a rule: Host Port 8080, Guest Port 80.

🚀 5. Install Nginx

Update Ubuntu packages 
sudo apt update
sudo apt upgrade -y
Install Nginx 
sudo apt install nginx -y
Start and enable Nginx 
sudo systemctl start nginx
sudo systemctl enable nginx
sudo systemctl status nginx
Allow HTTP through firewall if UFW is enabled 
sudo ufw allow 'Nginx HTTP'
sudo ufw status

Test Nginx by visiting http://VM_IP_ADDRESS or http://localhost:8080 if using NAT port forwarding.

🧪 6. Create a Simple Flask App

Install Python tools 
sudo apt install python3 python3-pip python3-venv -y
Create project folder 
sudo mkdir -p /var/www/flaskapp
sudo chown -R $USER:$USER /var/www/flaskapp
cd /var/www/flaskapp
Create and activate virtual environment 
python3 -m venv venv
source venv/bin/activate
Install Flask and Gunicorn 
pip install flask gunicorn
Create app.py 
from flask import Flask
from datetime import datetime

app = Flask(__name__)

@app.route("/")
def home():
    return f"""
    <html>
      <head>
        <title>Flask on Ubuntu + Nginx</title>
        <style>
          body {{
            font-family: Arial, sans-serif;
            background: #0f172a;
            color: white;
            text-align: center;
            padding-top: 80px;
          }}
          .card {{
            display: inline-block;
            background: #1e293b;
            padding: 32px;
            border-radius: 18px;
            box-shadow: 0 20px 50px rgba(0,0,0,0.35);
          }}
          h1 {{ color: #38bdf8; }}
        </style>
      </head>
      <body>
        <div class="card">
          <h1>Hello from Flask!</h1>
          <p>Served by Gunicorn behind Nginx.</p>
          <p>Server time: {datetime.now().strftime('%Y-%m-%d %H:%M')}</p>
        </div>
      </body>
    </html>
    """

if __name__ == "__main__":
    app.run(host="0.0.0.0", port=5000)
Test Flask directly 
python app.py

Open http://VM_IP_ADDRESS:5000. Stop the test server with Ctrl + C.

🦄 7. Run Flask with Gunicorn

Test Gunicorn 
cd /var/www/flaskapp
source venv/bin/activate
gunicorn --bind 127.0.0.1:8000 app:app

Gunicorn now serves Flask locally on port 8000. Nginx will proxy web traffic to this internal port.

🔁 8. Configure Nginx as a Reverse Proxy

Create Nginx site config 
sudo nano /etc/nginx/sites-available/flaskapp
Nginx config 
server {
    listen 80;
    server_name _;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
Enable the site 
sudo ln -s /etc/nginx/sites-available/flaskapp /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl reload nginx

⚙️ 9. Create a systemd Service for Flask

Create service file 
sudo nano /etc/systemd/system/flaskapp.service
systemd service 
[Unit]
Description=Gunicorn instance for Flask app
After=network.target

[Service]
User=YOUR_UBUNTU_USERNAME
Group=www-data
WorkingDirectory=/var/www/flaskapp
Environment="PATH=/var/www/flaskapp/venv/bin"
ExecStart=/var/www/flaskapp/venv/bin/gunicorn --workers 3 --bind 127.0.0.1:8000 app:app

[Install]
WantedBy=multi-user.target

Replace YOUR_UBUNTU_USERNAME with your actual Ubuntu username.

Enable and start service 
sudo systemctl daemon-reload
sudo systemctl start flaskapp
sudo systemctl enable flaskapp
sudo systemctl status flaskapp

🎉 10. Test the Final Website

Visit your app from the host machine:

Bridged network URL 
http://VM_IP_ADDRESS
NAT port forwarding URL 
http://localhost:8080

Useful troubleshooting commands

sudo systemctl status nginx
sudo systemctl status flaskapp
sudo journalctl -u flaskapp -n 50 --no-pager
sudo nginx -t
curl http://127.0.0.1:8000
curl http://localhost

Done: Ubuntu is running in VirtualBox, Nginx is installed, Flask is served by Gunicorn, and Nginx proxies requests to your app.
